E) Technical Fixes Backlog (Dev-Ready)
Technical SEO provides the foundational infrastructure that allows search engine crawlers to efficiently discover, parse, render, and index the optimized content. A compromised technical foundation will neutralize all subsequent content and local optimization efforts. Assumption explicitly flagged: Given that the URLs for Alberton Battery Mart's sitemap and specific product schemas were inaccessible during the initial crawl 17, the following technical backlog addresses the most probable structural bottlenecks inherent to modern automotive e-commerce CMS environments.
Priority 1: Faceted Navigation Index Bloat and Parameter Handling
* The Issue: Automotive websites utilizing "Find My Battery" faceted filters (Make/Model/Year permutations) frequently generate infinite, low-value crawl spaces via dynamically generated URL parameters (e.g., ?make=toyota&model=hilux&year=2021&engine=2.8gd6). When Googlebot encounters millions of these permutations, it exhausts the site's crawl budget, leaving core commercial pages unindexed.
* The Dev Implementation: The development team must implement strict, self-referencing rel="canonical" tags pointing back to the core product or primary category page for all filtered results. Furthermore, the robots.txt file must be aggressively configured to disallow crawling of dynamic search strings and faceted sorting parameters (e.g., Disallow: /*?make=*, Disallow: /*?sort=*).
* Validation Method: Continuous monitoring of the Google Search Console (GSC) "Crawled - currently not indexed" and "Discovered - currently not indexed" reports to confirm a steep reduction in parameterized URL discovery over a 14-day trailing period.
Priority 2: Core Web Vitals (CWV) and Mobile Render Speed Optimization
* The Issue: The entire emergency search intent strategy relies on capturing users stranded on the roadside, likely accessing the web via degraded 3G or 4G mobile networks. Heavy JavaScript (JS) execution payloads or unoptimized hero imagery will cause catastrophic Largest Contentful Paint (LCP) delays, resulting in massive bounce rates before the "Call Now" button even renders.
* The Dev Implementation: The architecture must enforce server-side rendering (SSR) or static site generation (SSG) for the critical rendering path, especially for the above-the-fold content on all emergency service pages. All product and hero imagery must be aggressively compressed utilizing next-generation formats (WebP or AVIF) and explicitly served with width and height attributes to prevent Cumulative Layout Shift (CLS). Non-critical third-party JavaScript (such as GA4 tags, live chat widgets, or Facebook pixels) must be strictly deferred or asynchronously loaded until after the First Contentful Paint (FCP) is achieved.
* Validation Method: The objective is to achieve a consistent Google PageSpeed Insights mobile score exceeding 85+, with an LCP strictly under 2.5 seconds, and a CLS score of 0.0 on simulated 4G throttling.
Priority 3: JavaScript Hydration and Internal Linking Architecture
* The Issue: Modern front-end frameworks (like React or Vue) often rely on client-side JavaScript routing for primary navigation. If search engine crawlers fail to execute this JS payload effectively (hydration failure), they will be completely unable to map the site hierarchy, leaving deep product pages orphaned.
* The Dev Implementation: The development team must guarantee that the primary DOM contains all critical internal links utilizing standard, parseable HTML <a href="..."> anchor tags. The hub-and-spoke architecture must be entirely navigable with JavaScript completely disabled.
* Validation Method: Disable JavaScript execution in Chrome DevTools and verify that all localized geographic spoke pages and nested product categories remain fully accessible and visible via the primary header and footer navigation menus.
Priority 4: Canonicalization and Duplicate Template Control
* The Issue: As the strategy scales out geographic service pages (Alberton, Meyersdal, Brackenhurst), there is an inherent risk of algorithmic cannibalization if the content footprint is too similar, prompting Google to arbitrarily select one page and suppress the remainder of the cluster.
* The Dev Implementation: Ensure the canonical tag on each individual suburb page strictly and accurately self-references its own unique URL. Enforce a programmatic character count threshold of 400 words of unique, localized text per location page to satisfy algorithmic quality thresholds.
* Validation Method: Execute a comprehensive site crawl utilizing Screaming Frog SEO Spider to programmatically identify missing, duplicated, or conflicting canonical instructions across the localized hub.
